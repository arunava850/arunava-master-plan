# 100 days project — overview

## Why this works for your specific goals

A 100 day project gives you a story with a beginning, middle, and end — the most powerful content format that exists. People don't just follow posts. They follow journeys.

| Benefit | What it means for you |
|---|---|
| Authority | 100 builds = undeniable proof. You didn't just study AI — you shipped 100 things with it. |
| Consistency | You never wonder what to post. Every day is a build, every build is a post. |
| Discovery | #100DaysOfBuilding already has an audience. You're joining a stream, not starting from silence. |
| Research | By day 30 you'll know which builds get 10x more engagement — those become your courses and tools. |

## What it is

A public commitment to build 100 small, useful things — documenting every single one across your website, LinkedIn, X, Instagram, and YouTube.

## What counts as one "build"

- A working script or tool (even 50 lines of Python)
- A data finding with a chart
- A new feature or automation
- A working prototype — rough but functional
- A Cursor prompt template or workflow
- A dashboard or visualisation
- A website feature or component

**Rule:** It must be real and runnable. Not a plan. Something you can show working.

## Days 1–10 — AI builds (establish builder identity)

- **Day 1:** Announcement post + one past build documented
- **Days 2–3:** Document best past AI build (debasreedeyart.com or Ainsemble)
- **Days 4–5:** AI content tool for artists (Claude API — generates captions, course descriptions)
- **Days 6–7:** Cursor workflow template — free resource for dev audience
- **Days 8–10:** Build log generator — paste notes, get a LinkedIn post draft

## Days 11–20 — Cricket analytics begins publicly

- **Day 11:** "I downloaded my first cricket dataset" — what's inside it
- **Days 12–14:** First data exploration — one surprising finding per day with a chart
- **Days 15–17:** First visualisation dashboard — matplotlib or plotly
- **Days 18–20:** First prediction attempt — and why it's wrong in interesting ways

## Days 21–30 — Artist + web tools

- **Days 21–23:** Artist website checklist app — "Is your site ready to sell?"
- **Days 24–26:** The debasreedeyart.com build story — full behind the scenes
- **Days 27–30:** Artist website starter template or kit

## Days 31–40 — AI builds (deeper)

- **Days 31–33:** Build log generator v2 — improved based on feedback
- **Days 34–36:** Cursor rules template library — shareable, high value for dev audience
- **Days 37–40:** AI automation for a specific workflow (document what you automate at Ainsemble — share what's appropriate)

## Days 41–60 — Cricket analytics (main build phase)

- **Days 41–45:** T20 win probability model v1 — first working version
- **Days 46–50:** Player performance across formats — which batters survive the transition
- **Days 51–55:** IPL player valuation model — auction value vs actual output
- **Days 56–60:** First interactive cricket dashboard — plotly or streamlit

## Days 61–70 — AI tools (productisation phase)

- **Days 61–64:** Polish the best AI tool from days 1–40 into something shareable
- **Days 65–67:** Build something for the artist audience — AI social media toolkit
- **Days 68–70:** Document the Ainsemble AI stack (what you can share)

## Days 71–90 — Cricket analytics (advanced)

- **Days 71–75:** Win probability model v2 — improved with more features
- **Days 76–80:** Head-to-head matchup tool — batter vs bowler historical stats
- **Days 81–85:** Pitch condition analyser — how surface affects selection
- **Days 86–90:** IPL auction predictor — build before next auction window

## Days 91–100 — Wrap up + launch

- **Days 91–95:** Artist website template kit — polished and free to download
- **Days 96–98:** Build the /100days showcase page on your website
- **Days 99–100:** Day 100 celebration post + what comes next — course announcement
