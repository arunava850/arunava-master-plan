## Prioritisation framework

When deciding what to build next, ask:
1. Does it serve one of my 4 audiences?
2. Can I build it in a weekend or less?
3. Will I learn something useful doing it?
4. Is there a real person who would use this?

If 3 of 4 are yes → build it.
